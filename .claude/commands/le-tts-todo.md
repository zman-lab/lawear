# le-tts-todo — TTS 변환 진행 현황 관리

> Case 단위로 TTS 변환 진행 상태를 추적.
> Lv.1/2/3은 묶어서 1건으로 관리.
> 아규먼트 없음 — 호출하면 현황 표 출력.

## 호출 시 동작

1. `docs/tts_progress.md` 읽기
2. 과목별 요약 + 전체 진행률 출력
3. 다음 작업 자동 제안 (우선순위 기반)

### 출력 형식

```
## TTS 변환 진행 현황

| 과목 | 전체 | 완료 | 진행중 | 미착수 | 진행률 |
|------|------|------|--------|--------|--------|
| 민소 | 12   | 0    | 0      | 12     | 0%     |
| 민법 | 8    | 0    | 0      | 8      | 0%     |
| ...  |      |      |        |        |        |

### 다음 작업
> 민소 예비 미케01 Case01 (미착수)
```

## 데이터 파일: `docs/tts_progress.md`

### 구조

```markdown
# TTS 변환 진행 현황

> 마지막 스캔: {날짜}
> 전체: {N}건 | 완료: {N} | 진행중: {N} | 미착수: {N}

## 민사소송법

| ID | PDF 파일 | Case | 상태 | 게시판 | 비고 |
|----|----------|------|------|--------|------|
| 2026_minso_yebi_mike01_01 | 민소_예비_미케_01.pdf | Case01 | 미착수 | | |
| 2026_minso_yebi_mike01_02 | 민소_예비_미케_01.pdf | Case02 | 미착수 | | |

## 민법
...
```

### 상태값

| 상태 | 의미 |
|------|------|
| 미착수 | PDF 미변환 |
| 진행중 | Lv.1/2/3 변환 작업 중 |
| 리뷰대기 | 게시판에 올려서 사용자 확인 대기 |
| 완료 | 사용자 OK + 커밋됨 |

### ID 체계

`project_id_system.md` 준수:
- `{yyyy}_{과목영문}_{단계영문}_{문번}_{부수문번}`
- 예: `2026_minso_yebi_mike01_01`

### 상태 갱신 시점

- **미착수 → 진행중**: TTS 변환 시작 시 (le-dev 워크플로우 내)
- **진행중 → 리뷰대기**: 게시판에 Lv.1/2/3 + diff 올렸을 때
- **리뷰대기 → 완료**: 사용자 OK 후 커밋 시
- **갱신 주체**: le-dev 메인 에이전트가 TTS 작업 중 직접 갱신

### PDF 파일 분류 규칙

**TTS 대상 (문+답 쌍):**
- `*_문.pdf` + `*_답.pdf` 쌍 (형법/형소)
- `*_미케_NN.pdf` 합본 (민법/민소 — 문+답 포함)
- `*_모고_NN.pdf` 합본 (형소 모의고사 — 문+답 포함)
- 2025 합격 모고 (강사별 — 문+답 포함)
- 2순환/3순환 파일 (과목별)

**TTS 제외:**

### Case 파싱

- PDF 내부의 Case/문제 구분은 PDF를 열어봐야 알 수 있음
- 초기 스캔 시: PDF 파일 단위로 1건 등록 (Case 수 미정)
- TTS 변환 시작 시: PDF 열어서 Case 수 확인 → 행 분할

### 스캔 절차

```
1. ~/Downloads/2026_USB/ 전체 PDF 탐색
2. 제외 패턴 필터링
3. 과목/단계 분류
4. docs/tts_progress.md 생성
5. 기존 ttsData.ts 대조 → 이미 변환된 것은 "완료" 표시
```

## 리스캔

사용자가 대화에서 "리스캔해줘", "PDF 추가했어" 등 요청 시:

```
1. ~/Downloads/2026_USB/ 재스캔
2. 기존 docs/tts_progress.md와 diff
3. 새로 발견된 PDF → "미착수"로 추가
4. 삭제된 PDF → 사용자에게 보고 (자동 삭제 안 함)
5. 기존 상태는 유지
```

## 우선순위 (다음 작업 제안)

CLAUDE.md의 과목별 우선순위 준수:

```
1순위: 논술 5과목
  민소 예비 → 민법 예비 → 형법 예비 → 형소 예비 → 부등법 2순환
2순위: 민사서류
3순위: 부등서류 → 보류
```

같은 과목 내에서는:
- 미케(미니케이스) > 모고(모의고사) > 순환
- 번호순 (01 → 02 → ...)

## le-dev 연동

- TTS 변환 작업 시작/완료 시 le-dev가 자동으로 상태 갱신
- `/le-tts-todo` 호출 → 현황 확인 → 다음 작업 선택 → `/le-dev`로 변환 시작
- 순환: `/le-tts-todo` → 작업 선택 → TTS 변환 → 상태 갱신 → `/le-tts-todo`
