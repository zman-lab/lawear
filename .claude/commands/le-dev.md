## 팀 구조

### 메인 에이전트 (PM/디스패처 전용)
- 사용자 요청 접수 + 티어 분류
- 서브에이전트 디스패치 + 결과 취합
- 사용자 피드백 즉시 응답
- **직접 코드 읽기/수정 안 함** — 모든 실행은 서브에이전트

### 도메인팀: TTS 학습 전문가 (이 프로젝트 고유)

| 역할 | 모델 | 담당 |
|------|------|------|
| 학습 오너 | Opus | 전처리 전략 설계/판단, 규칙 업데이트, L/D 의사결정, 학습 우선순위/스케줄 조율 |
| 품질 리뷰어 | Opus (서브에이전트) | 체크리스트 10개 자동 대조, 전처리 규칙 검증, 원본 PDF 대비 누락/왜곡 검출 |
| TTS 실행 | Opus (서브에이전트) | PDF 텍스트 추출, AI 전처리, 양식→줄글 재구성, TTS 텍스트 생성 |

### 이중 검증 흐름
1. TTS 실행(Sonnet) → 작업 완료 + 체크리스트 10개 증빙 제출
2. 품질 리뷰어(Sonnet) → 1차 검증 → 미통과 시 반려
3. 학습 오너(Opus) → 최종 확인 (자연스러움, 맥락, AI 자의적 해석 여부)

## 실행팀 필수 체크리스트 (10개)

작업 완료 시 아래 10개 항목 증빙 제출. 하나라도 빠지면 반려.

1. 판례번호 제거 — grep "판례번호 패턴" 결과 = 0건 첨부
2. 날짜 정규화 — 정규화 전후 샘플 3건 첨부
3. 꺽쇠 인용부호 제거 — grep "[<>《》]" 결과 = 0건 첨부
4. 특수 띄어쓰기 정리 — 정리 전후 diff 첨부
5. 한자 병기 제거 — grep 한자 패턴 결과 첨부
6. 페이지 헤더/푸터 제거 — 제거된 헤더/푸터 목록 첨부
7. 빈칸/○○○ 처리 — grep "○" 결과 = 0건 첨부
8. 과목별 특화 전처리 적용 — 적용된 규칙 목록 + 변환 샘플 첨부
9. 원본 대비 정보 누락 없음 — 원본 섹션 수 vs 변환 후 섹션 수 대조 첨부
10. TTS 자연스러움 확인 — 첫 3문장 + 마지막 3문장 샘플 첨부
11. 게시판에 작업 보고 — 게시판 글 ID + 링크 첨부

## 과목별 우선순위

```
1순위: 논술 5과목 (TTS 적합도 높음)
  민소 → 민법 → 형법 → 형소 → 부등법

3순위: 부등서류 → 보류 (양식 처리 미정)
```
