> 법무사 2차 시험 PDF → TTS 텍스트 변환 워크플로우.
> 세션 고정: 이 스킬 호출 후 모든 요청을 le-dev 워크플로우로 처리.
> 각 응답 끝에 `🟦 [le-dev] 세션 활성` 표시.

## 필수 선행 로드

이 스킬 호출 시 아래 서브 스킬을 즉시 체인 로드:
- `/context-guard` — 대형 작업 자동 실행 모드
- `/le-worktree` — 워크트리 규칙
- `/le-patrol` — 게시판 순찰

## 최우선 규칙: AI 자의적 해석 금지

- 원본 PDF에 있는 정보만 사용 — 내용 추가/변경/보충 절대 금지
- 기계적 변환(기호→텍스트, 한자→한글)은 허용
- 법률적 판단/해석 변경은 절대 금지
- 애매하면 원본 표현 유지
- **이 규칙은 다른 모든 규칙보다 우선**

## TTS 변환 워크플로우

```
1. 요청 접수 (과목/파일 지정)
2. 티어 분류 (S/M/L/D)
3. 워크트리 생성
   git -C /Users/nhn/zman-lab/lawear worktree add -b wt/{작업명} ../lawear-{작업명} main
4. PDF 텍스트 추출 (Phase 1)
   - PyMuPDF로 텍스트 레이어 추출
5. AI 전처리 (Phase 2)
   - tts_rules.md 규칙 적용 (기본 10개 + 추가 11개)
   - 과목별 특화 규칙 적용
6. TTS 텍스트 생성 (Phase 3)
   - 문제 → 목차 → 답안 순서
   - 호흡 조절 쉼표 삽입
   - 특수기호 최종 제거
7. diff 생성 (원본 vs TTS)
8. QA (듣는 사람 입장 검증)
9. 실행팀 체크리스트 10개 증빙 제출
10. 품질 리뷰어 1차 검증
11. 오너 최종 확인
12. 커밋 + 워크트리 정리
13. 게시판 보고
```

## 실행팀 필수 체크리스트 (10개)

작업 완료 시 아래 10개 항목 증빙 제출. 하나라도 빠지면 반려.

1. 판례번호 제거 완료
2. 날짜 정규화 완료 (상대적 시간 표현 포함)
3. 꺽쇠 인용부호 제거 완료
4. 특수 띄어쓰기 정리 완료
5. 한자 병기 제거 완료 (인명 甲乙丙은 한글 변환)
6. 페이지 헤더/푸터 제거 완료
7. 빈칸/○○○ 처리 완료
8. 과목별 특화 전처리 적용 완료
9. 원본 대비 정보 누락 없음 확인
10. TTS로 읽었을 때 자연스러운 문장 확인

## 과목별 우선순위

```
1순위: 논술 5과목 (TTS 적합도 높음)
  민소 → 민법 → 형법 → 형소 → 부등법
2순위: 민사서류 (서술형 부분)
3순위: 부등서류 → 보류 (양식 처리 미정)
```

## 게시판 소통

- **author**: 세션 ID (예: lawear-7c1e)
- **보고 300자 규칙**: 300자 넘으면 게시판 글 → 링크만 전달
- **톤**: 수다 톤 ("민소 미케01 Case1~3 변환 끝! 판례번호 제거가 의외로 까다로웠음 ㅋ")
- **사용자 피드백 확인 습관**: 작업 전 관련 게시글 댓글 반드시 확인
