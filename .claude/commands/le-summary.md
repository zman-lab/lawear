# le-summary — 핵심요약 TTS 변환 스킬

> Lv.2 핵심요약 TTS 변환.
> 입력: Lv.1 데이터 (문제 간결 + 답안 원문 클린)
> 출력: A~F 구조 재구성 답안
>
> A~F 구조 (#756 확정):
> - A. 쟁점. (예: "제33조 국선변호인 제3항, 재량적 국선변호인 선정.")
> - B. 결론. 간결하게 ("위법하다." 수준). 장문 금지.
> - C. 이유. 원문 핵심 법리만, AI 보충 금지 (R-09)
> - D. 조문. 조문명+제목
> - E. 판례. 자연어 서술 ("판례 세 가지가 있다. 첫째,..."). 목록 나열 X.
> - F. 사안의 포섭. 핵심만. 원문 복사 X.
> - 전체 6문장 이내 목표.
> - 목차: 핵심 쟁점 위주 정리. 주어/대상 생략 금지.

## 언제 사용하나

- 이미 le-dev로 빠른복습 Lv.1 TTS를 만든 뒤
- 시험 직전 짧게 돌려볼 "요약본"이 필요할 때
- 30분짜리를 10~15분으로 줄이고 싶을 때

## 입력

- 기존 Lv.1 TTS 텍스트 파일 (또는 원본 PDF 직접 지정)
- 예: `le-summary 민소_미케01_Case1.txt`

## 출력 형식

```
[목차]
핵심 쟁점 위주 정리 (주어/대상 생략 금지)

[답안]
A. 쟁점.
B. 결론.
C. 이유.
D. 조문.
E. 판례.
F. 사안의 포섭.
```

- 문제: Lv.1과 동일 수준 유지
- 목차 → 답안(A~F) 순서
- diff (원본 Lv.1 vs le-summary 결과)
- QA (듣는 사람 기준 자연스러움 확인)

## 규칙

### 기본: le-dev 규칙 + tts_rules.md 전체 준수

R-01~R-22 모두 적용. 충돌 시 R-09(자의적 해석 금지)가 최우선.

### le-summary 추가 축약 규칙 (R-01~R-22보다 더 공격적)

**S-01. 문제 섹션 완전 제거**
- "문제", "사례", "물음", "질문" 포함 블록 전체 삭제
- 단, 문제 안에만 있던 고유 키워드가 답안에도 없으면 답안 앞에 1줄 요약 삽입

**S-02. 답안 과감 압축**
- 법조문 인용은 조문 번호만 ("민법 제750조" 유지, 조문 본문은 제거)
- 판례 내용 요약 → 결론 1문장만 (판례번호는 이미 제거되어 있음)
- "따라서", "왜냐하면", "이에 따라" 등 연결어 대부분 제거
- 소결론/중간결론 반복 제거 (마지막 결론만 유지)

**S-03. 중복 표현 제거**
- 같은 법률 개념이 2회 이상 설명되면 첫 번째만 유지
- "~라고 할 것이다", "~라고 보아야 할 것이다" → "~이다"
- "~에 해당한다고 할 수 있다" → "~에 해당한다"

**S-04. 목차는 원문 그대로**
- 목차 축약/재구성 금지
- 단, 목차 항목 중 답안에서 완전히 제거된 내용은 해당 항목도 제거

**S-05. 정보 누락 금지선**
- 법적 요건, 효과, 예외는 절대 제거 금지
- 삭제 기준: "이게 없으면 법적 판단이 달라지는가?" → 달라지면 유지

## 워크플로우

```
1. 입력 파일 확인 (Lv.1 TTS 또는 원본 PDF)
2. 문제 섹션 제거
3. 답안 압축 (S-02~S-04 적용)
4. tts_rules.md R-01~R-22 재적용 (기계적 검증)
5. diff 생성 (입력 대비 제거/압축 내역)
6. QA: "이 텍스트만 들어도 시험 답 쓸 수 있나?"
7. 체크리스트 10개 확인 (le-dev 기준 동일)
8. 완료 보고
```

## 입력값

$ARGUMENTS
