# le-supersimple — 슈퍼심플 TTS 변환 스킬

> 판례 못 외웠을 때 기본 점수용. 의의/요건/효과 키워드만 단문 나열.

## 언제 사용하나

- 판례/논증 구조는 포기하고 기본 개념 점수만 챙길 때
- 시험 당일 아침, 5분 안에 전과목 훑을 때
- le-summary보다 더 짧게 — 개념 키워드만 필요할 때

## 입력

- 원본 PDF 또는 기존 TTS 파일 (Lv.1 또는 le-summary 결과 모두 가능)
- 예: `le-supersimple 민소_미케01_Case1.txt`

## 출력 형식

```
[제목]. 의의: ~. 요건: 1) ~, 2) ~, 3) ~. 효과: ~. (예외: ~.)
```

- 문제/목차/답안 형식 없음
- 각 쟁점을 1~3문장 단문으로 나열
- 쟁점 간 구분은 줄바꿈 1개
- diff 없음 (원본과 형식이 너무 달라 무의미)
- QA: "이 키워드 들으면 빈칸 채울 수 있나?"

## 규칙

### 기본: tts_rules.md R-01~R-22 적용 (기계적 규칙만)

- R-09(자의적 해석 금지) 최우선
- 법률 용어는 정확히 — 개념 자체를 바꾸지 않는다

### le-supersimple 전용 규칙

**SS-01. 추출 항목 (있는 것만)**
- 의의: 무엇인가 (1문장)
- 취지: 왜 있는가 (필요 시만, 선택)
- 요건: 번호 매겨 나열 (1), 2), 3)...)
- 효과: 어떻게 되는가
- 예외: 있으면 "(예외: ~)" 괄호로 추가
- 판례: 결론 키워드 1개만 (예: "실질적 지배 기준 적용")

**SS-02. 문장 규칙**
- 주어 생략 가능 ("~은 ~이다" → "~이다")
- 조사 최소화
- "~라고 할 것이다", "~에 해당한다고 볼 수 있다" 전부 → "~이다"
- 접속사, 연결어 전부 제거

**SS-03. 제목 규칙**
- 원본 쟁점/소제목 그대로 사용
- 괄호 표현은 제거 (예: "불법행위(요건과 효과)" → "불법행위")

**SS-04. 정보 누락 금지선**
- 요건 개수 변경 금지 (원본이 4개면 4개)
- 효과/예외 생략 금지
- 숫자/기간/비율은 반드시 유지 (예: "30일", "3분의 2")

**SS-05. 자의적 해석 금지 (R-09 재확인)**
- 원본에 없는 요건/효과 추가 금지
- "일반적으로", "보통" 등 임의 수식어 추가 금지

## 워크플로우

```
1. 입력 파일에서 쟁점 목록 추출
2. 각 쟁점별 SS-01 항목 추출
3. SS-02~SS-03 단문 변환
4. tts_rules.md 기계적 규칙 적용 (판례번호 제거 등)
5. QA: "이 키워드들로 답안 뼈대를 잡을 수 있나?"
6. 완료 보고 (체크리스트 생략 — 형식 자체가 다름)
```

## 입력값

$ARGUMENTS
