## 현재 규칙 구조

### 기본 전처리 규칙 (10개)

| # | 규칙명 |
|---|--------|
| 1 | 한자 인명 → 한글 (甲→갑, 乙→을, 丙→병) |
| 2 | 판례번호 완전 제거 |
| 3 | 날짜 정규화 (YYYY. M. D. → YYYY년 M월 D일) |
| 4 | 페이지 헤더/푸터 제거 |
| 5 | 꺽쇠 인용부호 제거 (「」『』) |
| 6 | 특수 띄어쓰기 정리 |
| 7 | 한자 병기 제거 (인명 甲乙丙은 한글 변환) |
| 8 | 원문자 번호 통일 (① → 첫째 또는 1번) |
| 9 | O/x 표기 변환 (경우-O → 인정됨, 경우-x → 부정됨) |
| 10 | 빈칸(○○○) 처리 |

### 추가 규칙 (R-01~R-11)

| 번호 | 규칙명 | 수정 가능 |
|------|--------|----------|
| R-01 | 판례번호는 완전 제거 (변환 아님) | O |
| R-02 | 날짜 → 상대적 시간 표현 활용 | O |
| R-03 | 괄호(A(B)) → "A 혹은 B" | O |
| R-04 | 호흡 조절 쉼표 삽입 | O |
| R-05 | 콜론(:) → 쉼표(,) | O |
| R-06 | 문제 후 목차 먼저 → 답안 | O |
| R-07 | 목차는 원문 그대로 | O |
| R-08 | 특수기호 제거 ([] — : () /) | O |
| R-09 | **AI 자의적 해석 금지 — 원본 정보만 사용** | **수정 불가** |
| R-10 | diff 필수 제공 | O |
| R-11 | 듣는 사람 입장 QA 필수 (5개 항목) | O |

## 규칙 적용 우선순위

```
1. 게시판 #462 피드백 (tts_rules.md보다 우선 — 최신 확정본)
2. R-09 (자의적 해석 금지) — 최상위. 다른 규칙과 충돌 시 이 규칙이 우선
3. R-06, R-07 (목차 구조) — 답안 구성의 뼈대
4. R-01~R-05, R-08 (텍스트 정제) — 기계적 적용
5. R-04 (호흡 조절) — 최종 단계에서 듣기 관점 조정
6. R-10, R-11 (검증) — 출력 후 필수
```

## 규칙 추가 워크플로우

```
1. 추가 요청 접수 (사용자 피드백 기반)

2. 충돌 검토
   - 기존 규칙 중 동일/유사 항목 있는지 확인
   - R-09와 충돌 여부 확인 (원본 정보 변형이면 거부)
   - 우선순위 상 어디에 위치할지 판단

3. 워크트리 확인/생성
   git -C /Users/nhn/zman-lab/lawear worktree list
   → 기존 워크트리 없으면:
   git -C /Users/nhn/zman-lab/lawear worktree add -b wt/rules-update ../lawear-rules-update main

4. tts_rules.md 수정
   - 새 규칙 번호 할당 (R-12, R-13... 순서로)
   - 규칙명, 설명, 예시 추가
   - 우선순위 섹션 업데이트

5. 변경 사유 기록
   - 규칙 하단에 변경 이력 추가:
     > [날짜] R-XX 추가: {사유} (게시판 #{글번호} 피드백)

6. 커밋
   git -C {워크트리경로} add docs/handover/tts_rules.md
   git -C {워크트리경로} commit -m "[근형] TTS 규칙 추가: R-XX {규칙명}"

7. 게시판 변경 내역 공유
   - lawear-work 게시판에 변경 내역 포스팅 (Haiku로 위임)
```

## 규칙 수정 워크플로우

```
1. 수정 요청 접수 (규칙 번호 + 수정 내용)

2. R-09 수정 시도 → 즉시 거부
   "R-09(자의적 해석 금지)는 최상위 규칙으로 수정/삭제 불가합니다."

3. 기존 규칙과 충돌 여부 확인
   - 수정 후 다른 규칙과 모순이 생기는지 검토
   - 충돌 발견 시 사용자에게 선택지 제시

4. 우선순위 변경 필요 여부 판단
   - 수정 내용이 우선순위 변경을 요구하면 함께 업데이트

5. 워크트리 확인/생성 (추가와 동일)

6. tts_rules.md 수정 + 변경 사유 기록

7. 커밋 + 게시판 공유 (추가와 동일)
```

## 규칙 변경 시 필수 사항

```
□ 변경 사유 기록 (날짜 + 사유 + 참조 게시글 번호)
□ 기존 규칙과의 충돌 여부 확인
□ 게시판에 변경 내역 공유 (다른 세션이 알 수 있도록)
□ 커밋 메시지에 규칙 번호 명시
```

## 과목별 특화 규칙 관리

```
과목별 특화 규칙은 tts_rules.md의 "과목별 특화 전처리" 섹션에서 관리.
추가/수정 시 동일한 워크플로우 적용.

| 과목 | 현재 특화 규칙 |
|------|--------------|
| 민소/민법 | 빈칸 박스 처리 전략 (선택 필요) |
| 형법/형소 | ★★ → "중요", 중첩 괄호 단순화 |
| 부등법 | "명문x" → "명문 규정 없음" |
| 민사서류 | 표/양식 → AI 줄글 재구성 |
| 부등서류 | 보류 (미정) |
```

