# 샘플 출력 형식 가이드

TTS 변환 결과물의 표준 출력 형식. 모든 과목에 동일 적용.

## 출력 구조

```
문제
  ↓
목차
  ↓
답안 (목차 순서대로)
  ↓
diff (원본 vs TTS)
  ↓
QA (듣는 사람 입장 검증)
```

## 각 섹션 상세

### 문제

- 키워드 포함 요약 (전문 그대로가 아닌 핵심 키워드 중심)
- 답안 구성에 필요한 키워드는 반드시 포함
  - 예: 채권자대위 문제 → "피보전채권의 존재", "보전의 필요성" 등 답안 키워드가 문제에 드러나야 함
- 날짜는 상대적 시간 표현 활용
  - 예: "2000년 1월 1일 대여, 2005년 3월 소장 접수" → "2000년 1월경 영업준비자금 명목으로 대여하고 5년이 지난 이후 소장 접수"
- 문제의 맥락을 알 수 있게 구성

### 목차

- 원문 그대로 (요약/재구성 금지)
- 문제 읽은 직후 바로 목차를 읽어줌
- 목차를 먼저 들으면 답안 구조가 머릿속에 잡힘

### 답안

- 목차 순서에 맞게 구성
- 2차 시험 답안 제출 시 핵심이 되는 부분 위주
- 호흡 조절 쉼표 적절히 삽입
- 괄호 → "혹은" 변환
- 콜론 → 쉼표 변환
- 판례번호 완전 제거
- 특수기호 제거

### diff

- 원본 텍스트와 TTS 변환 텍스트의 비교
- 변경된 부분을 명확히 표시
- 사용자가 최종 검수용으로 활용
- 형식 예시:

```
원본: 행위시법주의(소급효금지의 원칙)
TTS:  행위시법주의 혹은 소급효금지의 원칙

원본: 결론: 타당하지 않다(부당하다).
TTS:  결론, 타당하지 않다 혹은 부당하다.

원본: (대판 2018.7.19. 2018다22008)
TTS:  (삭제)
```

### QA

듣는 사람 입장에서 아래 항목을 자체 검증:

1. 이 부분이 구분되어 들리는가?
2. 문장이 너무 길어서 맥락 이해나 암기가 힘들지 않은가?
3. 날짜/인명이 혼동되지 않는가?
4. 법률 용어가 정확히 전달되는가?
5. AI가 자의적으로 해석/보충한 부분이 없는가?

QA 결과에서 문제 발견 시 해당 부분을 수정 후 재제출.

## 출력 예시 (민사소송법)

```
문제,
갑은 을을 상대로 매매계약 해제에 따른 매매대금 반환청구의 소를 제기하여
승소 확정판결을 받았다.
약 11년이 지난 후 갑은 확정판결에 기한 채권의 시효중단을 위해
다시 소를 제기하였다.
을은 소멸시효 완성을 주장하였고,
법원은 승소확정판결과 동일한 소송물이므로
시효중단을 구할 이익이 없다며 직권으로 소를 각하하였다.
법원의 판단은 타당한가,

목차,
1 결론
2 근거, 확정판결에 의한 채권의 시효중단을 위한 재소의 이익
3 후소 법원의 심리 범위
4 사안의 적용

답안,
결론, 타당하지 않다.

근거, 확정된 승소판결에는 기판력이 있으므로 ...
(이하 목차 순서대로)
```
