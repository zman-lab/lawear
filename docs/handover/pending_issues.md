# 미해결 과제

## P-01. 부등서류(등기신청서) 양식 처리 — 보류

### 현황
- 부동산등기서류 과목의 등기신청서 양식은 표/2단배치/빈칸(○○○)이 혼재
- TTS 적합도 D등급 (양식 부분)
- 사용자 피드백: "이건 너무 심각하다... 일단 보류"

### 선택지
1. AI가 양식을 줄글로 요약 → TTS 변환 (정보 손실 가능)
2. 해설/첨부서면 설명 부분만 TTS → 양식은 스킵
3. 양식을 구조화된 서술로 재구성 (예: "신청서의 부동산 표시란에는 ...")
4. 완전 제외

### 결정 시기
- Phase 1(논술 5과목) 완료 후 재검토 예정

## P-02. AI 자의적 해석 vs 자연스러움 균형

### 현황
- R-09 규칙: AI 자의적 해석 금지, 원본 정보만 사용
- 하지만 TTS 자연스러움을 위해 문장 재구성이 필요한 경우 존재
- 두 요구사항이 충돌하는 영역이 있음

### 구체적 충돌 사례
- 원본: 축약된 법률 표현 (예: "명문x") → TTS: "명문 규정 없음"
  - 이건 자의적 해석인가, 단순 기호 변환인가?
- 원본: 날짜 나열 → TTS: 상대적 시간 표현
  - 시간 계산을 AI가 하는 것은 해석인가?

### 원칙
- 기계적 변환(기호→텍스트, 한자→한글)은 허용
- 원본에 없는 정보 추가는 금지
- 법률적 판단/해석 변경은 절대 금지
- 애매한 경우 원본 표현 유지

### 해결 방향
- 규칙을 "허용/금지" 이분법이 아닌 예시 기반으로 구체화
- 실제 변환 과정에서 발견되는 애매한 케이스를 규칙에 추가

## P-03. TTS 엔진 선택 미정

### 현황
- 어떤 TTS 엔진을 사용할지 아직 결정되지 않음

### 고려사항
- 한국어 음질 (법률 용어 발음 정확도)
- 비용 (590개 PDF × 다수 문제 = 대량)
- 속도 조절 (학습용이므로 약간 느린 속도가 적합할 수 있음)
- 쉼표/마침표에서의 호흡 자연스러움

### 후보
| 엔진 | 한국어 | 비용 | 비고 |
|------|--------|------|------|
| Google Cloud TTS | 좋음 | 유료 | WaveNet 한국어 지원 |
| CLOVA Voice | 우수 | 유료 | 네이버, 한국어 특화 |
| OpenAI TTS | 보통 | 유료 | 다국어, 한국어는 보통 |
| Edge TTS | 보통 | 무료 | Microsoft, 무료 사용 가능 |

### 결정 시기
- Phase 3 시작 전까지 결정 필요
- Phase 2까지는 텍스트 생성이므로 TTS 엔진 무관

## P-04. 밑줄/강조 복원

### 현황
- 전 과목에서 밑줄/색상 강조가 텍스트 추출 시 소실 (C등급)
- 수험생에게 강조 표시는 중요한 학습 정보

### 복원 방안
- PyMuPDF의 어노테이션 레이어 추출
  - `page.annots()` — 주석 레이어
  - `page.get_text("dict")` — 텍스트 스타일 정보 (bold, italic, underline)
- 추출된 강조 정보를 TTS에서 활용:
  - "중요" 접두어 삽입
  - TTS 속도/음량 변화 (엔진 지원 시)
  - 별도 마커로 표시

### 검증 필요
- 실제 PDF에서 PyMuPDF가 어노테이션을 얼마나 추출하는지 테스트 필요
- PDF 생성 방식에 따라 어노테이션이 텍스트 스타일인지 별도 레이어인지 다름

## P-05. 빈칸 처리 전략 미확정 (민소/민법)

### 현황
- 미케 자료에 빈칸 박스가 있음
- 빈칸을 유지할지, 정답을 삽입할지 결정 필요

### 선택지
1. 빈칸 유지: "빈칸" 또는 "(    )"으로 읽기 → 사용자가 머릿속으로 채움
2. 정답 삽입: 정답을 넣어서 완성된 문장으로 읽기 → 암기용
3. 두 버전 모두 생성: 테스트용 + 암기용

### 결정 시기
- 사용자에게 확인 필요

## P-06. 문제 요약 수준 가이드라인

### 현황
- R-02(상대적 시간)와 문제 요약을 적용하면 원본 대비 상당히 축약됨
- 어디까지 축약해도 되는지 명확한 기준 부재
- 답안 키워드는 반드시 포함해야 하지만 사실관계 세부사항은?

### 해결 방향
- 과목별 샘플 3~5개를 변환 후 사용자 피드백으로 수준 확정
- "이 정도면 문제 맥락을 알 수 있다" 기준선 설정
