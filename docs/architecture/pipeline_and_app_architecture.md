## v2 변경 요약

> **핵심**: 서버/빌드에서 Cloud TTS로 MP3 사전 렌더링하여 배포하던 구조를
> **TTS 텍스트 데이터만 배포하고, 사용자가 로컬에서 음성 선택 + MP3/WAV 렌더링**하는 구조로 전환.

### 무엇이 바뀌었나

| 항목 | v1 | v2 |
|------|----|----|
| 오디오 렌더링 | 서버/빌드 시 Cloud TTS | **사용자 기기에서 로컬 렌더링** |
| 배포물 | TTS 텍스트 + MP3 (~2.4GB) | **TTS 텍스트만 (수 MB)** |
| TTS 엔진 선택 | 개발팀이 결정 (Google Neural2) | **사용자가 선택** (기기 엔진, 음성) |
| TTS 비용 | 개발팀 부담 ($0~$23) | **사용자 부담 (무료 TTS도 가능)** |
| 재생 전략 | 듀얼 모드 (실시간/MP3) | **투명 전환** (캐시 있으면 MP3, 없으면 TTS 실시간) |
| Phase 4 | Cloud TTS 렌더링 파이프라인 | **제거** (로컬 렌더링으로 대체) |

### 역할 분담 (v2 변경)

| 우리(개발팀)가 하는 것 | 사용자가 하는 것 |
|----------------------|----------------|
| PDF -> TTS 텍스트 변환 (Opus 3인팀) | TTS 엔진/음성 선택 |
| TTS 텍스트 데이터 배포 (수 MB) | 로컬 MP3/WAV 렌더링 (선택) |
| 조문 매핑 (법령 API) | 배속/반복모드 설정 |
| 텍스트 버전 관리 | 오프라인 캐시 관리 |

### 1.3 각 Phase 상세

#### Phase 1: PDF 텍스트 추출

| 항목 | 내용 |
|------|------|
| 도구 | PyMuPDF (fitz) |
| 입력 | ~/Downloads/2026_USB/ (590 PDF) |
| 출력 | raw/{과목}/{파일}.txt |
| 자동화 | 100% (기계적 추출) |
| 소요 | ~5초/PDF, 전체 ~50분 |

```python
# 핵심 로직
import fitz

def extract_pdf(pdf_path: str) -> str:
    doc = fitz.open(pdf_path)
    text = ""
    for page in doc:
        text += page.get_text("text")
    return text
```

**1차 필터링**: 590개 중 TTS 대상(미케/모고/답안) 251개만 처리. 강의계획서, 쪽지시험 등은 제외.

#### Phase 2: AI TTS 전처리

| 항목 | 내용 |
|------|------|
| 도구 | Claude Opus (3인팀: 작가-수험생-판사) |
| 입력 | raw/ 텍스트 + TTS 규칙 R-01~R-18 |
| 출력 | tts/{과목}/{파일}/{case}.txt |
| 자동화 | 반자동 (Claude 세션에서 수동 실행) |
| 소요 | ~2분/문제, 전체 ~30시간 (900문제) |

**기계적 전처리 (rules.py로 자동화 가능)**:
- R-01: 한자 인명 변환 (甲->갑, 乙->을)
- R-02: 판례번호 제거 (정규식)
- R-03: 날짜 정규화 (2020. 1. 5. -> 2020년 1월 5일)
- R-04: 헤더/푸터 제거 (페이지 번호 패턴)
- R-05: 꺽쇠 인용부호 제거
- R-06: 특수 띄어쓰기 정리
- R-07: 한자 병기 제거
- R-14: 제+숫자 복합어 (제1심->제일심)
- R-16: 조문 제목 자동 삽입 (법령 API 캐시 활용)

**AI 판단 필요 (Opus 3인팀)**:
- R-02/R-17: 날짜->상대적 시간, 문맥 생략
- R-03: 괄호 변환 (보충설명 vs 동의어 판별)
- R-04/R-12: 호흡 쉼표 삽입, 문제 간결화
- R-15: 불필요 수식어 제거

#### 1.5a-1 TTS 엔진 자동 탐지 (v2 추가)

앱 초기화 시 기기에 설치된 모든 TTS 엔진을 자동 탐지하여 사용자에게 선택지 제공:

**플랫폼별 API**:
- Android: `TextToSpeech.getEngines()` → 설치된 TTS 엔진 목록 (Google, Samsung, Naver 등)
- iOS: `AVSpeechSynthesisVoice.speechVoices()` → 사용 가능한 음성 목록
- Web: `window.speechSynthesis.getVoices()` → 브라우저 음성 목록

**자동 탐지 흐름**:
1. 앱 시작 → 플랫폼 감지
2. 해당 플랫폼 API로 사용 가능한 엔진/음성 전체 조회
3. 한국어(ko-KR) 지원 음성만 필터링
4. 설정 화면에 목록 표시 (엔진명, 음성명, 성별, 샘플 재생 버튼)
5. 사용자 선택 → 설정 저장
6. 유료 TTS 앱을 사용자가 설치하면 다음 앱 시작 시 자동으로 목록에 추가됨

**인터페이스 확장**:

```typescript
interface TtsEngineAdapter {
  listEngines(): TtsEngine[]                               // (v2 추가) 설치된 엔진 목록
  listVoices(engineId?: string): Voice[]                   // 특정 엔진의 음성 목록 (미지정 시 전체)
  speak(text: string, voice: Voice): void
  renderToFile(text: string, voice: Voice, path: string): Promise<string>
}

interface TtsEngine {
  id: string          // 'com.google.android.tts', 'com.samsung.SMT' 등
  name: string        // 'Google', 'Samsung' 등
  installed: boolean
}
```

### 3.4 로컬 렌더링 용량 추정 (v2 변경)

```
과목별 (사용자가 선택 렌더링):
  민법:     250문제 x 5.5분 = ~690MB
  형법:     200문제 x 5.5분 = ~550MB
  민소:     180문제 x 5.5분 = ~500MB
  부등:     170문제 x 5.5분 = ~470MB
  형소:     100문제 x 5.5분 = ~275MB
```

| 전략 | 용량 | 장단점 |
|------|------|--------|
| 전체 렌더링 | ~2.4GB | 완전 오프라인, 초기 렌더링 시간 필요 |
| 과목별 렌더링 | ~500MB/과목 | 선택적, **권장** |
| 파일별 렌더링 | ~15MB/파일 | 세밀, UI 복잡 |
| TTS 실시간만 | 0 | 캐시 불필요, 배터리 소모 큼 |

**권장**: TTS 실시간 기본 + 과목별 선택 렌더링.
- 기본: TTS 실시간 재생 (설치 즉시 사용)
- 수동: "민소 전체 렌더링" 버튼 -> 백그라운드에서 MP3 생성
- 용량 표시: "민소 렌더링 (약 500MB)" UI에 명시
- WiFi 환경에서 자동 렌더링 옵션 (설정에서 ON/OFF)

### 3.5 배터리 소모 비교

| 방식 | CPU 사용 | 배터리 | 비고 |
|------|---------|--------|------|
| TTS 실시간 재생 | 높음 (실시간 합성) | 많이 소모 | 1시간 ~15% |
| 로컬 MP3 재생 (하드웨어 디코딩) | 극히 낮음 | 최소 | 1시간 ~3% |
| 로컬 MP3 + 화면 꺼짐 | 극히 낮음 | 최소 | 1시간 ~2% |
| 로컬 렌더링 중 | 높음 (TTS 합성 + 인코딩) | 많이 소모 | 백그라운드, WiFi+충전 시 권장 |

**결론**: 로컬 MP3 캐시 재생은 TTS 실시간 대비 배터리 소모 5~7배 절감. 렌더링은 충전 중에 하면 무방.

### 3.6 오디오 형식 — 사용자 선택 (v2 변경)

| 형식 | 음질 | 용량 | 권장 |
|------|------|------|------|
| MP3 64kbps | 충분 (음성) | ~2.4GB (전체) | **기본 권장** |
| MP3 128kbps | 우수 | ~4.8GB (전체) | 고품질 옵션 |
| WAV | 원본 | ~15GB+ (전체) | 비권장 (용량 과다) |

**앱 설정에서 선택 가능**. 기본값: MP3 64kbps.
