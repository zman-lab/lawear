| 부등서류 | 기찬 01 구분건물 | A(해설)/**C(양식)** | B- | C | **D (양식 부분)** |

## 등급 기준

- **A**: 완벽하게 추출, 구조 보존
- **B**: 대부분 추출, 경미한 정리 필요
- **C**: 소실 또는 추출 불가
- **D**: TTS 부적합, 별도 처리 필요

## 핵심 발견사항

### 1. OCR 불필요 — 전 과목 텍스트 임베딩
모든 590개 PDF가 텍스트 레이어를 포함. PyMuPDF로 직접 추출 가능.

### 2. 밑줄/색상 강조 전 과목 소실
텍스트 추출 시 밑줄, 형광펜, 색상 강조가 모두 소실됨. PyMuPDF 어노테이션 레이어를 별도로 추출하면 복원 가능성 있음 (미검증).

### 3. 서술형 자료 = TTS 최적
논술 5과목과 민사서류 소장/해설은 자연어 서술 형식 → 전처리 규칙만 적용하면 TTS 입력 텍스트로 바로 사용 가능.

### 4. 부등서류 양식 = 유일한 난관
등기신청서 양식 부분은 표/2단배치/빈칸(○○○)이 섞여 있어 직접 TTS 변환이 비효율적. 사용자 피드백: "이건 너무 심각하다... 일단 보류". → Phase 1에서 제외, 추후 별도 처리.

## 과목별 특성 정리

### 민사소송법
- 판례 중심 서술, 사실관계 + 쟁점 + 결론 구조
- 날짜/인명이 빈번 → 상대적 시간 표현 + 한자 인명 변환 필수
- 복잡한 절차적 쟁점 → 목차 선행 읽기가 효과적

### 민법
- 사실관계가 길고 날짜/금액이 다수 등장
- 청구권 경합, 요건사실 정리 등 논리적 서술
- 조문 번호 빈번 → "(제598조, 제603조)" 처리 필요

### 형법
- 법률 개정 전후 비교가 핵심
- 별표(★★) 중요도 표시 → "중요" 텍스트 변환
- 중첩 괄호 다수 → 단순화 필요

### 형사소송법
- 판례 + 조문 혼합 서술
- 사건 경과가 시간순 → 날짜 정리 중요
- 필요적/재량적 구분 등 법률 용어 정확성 필수

### 부동산등기법
- 서술형 답안으로 TTS 적합도 높음
- 학설/판례 대비, 효과/범위 등 체계적 서술
- "명문x" → "명문 규정 없음" 변환 등 과목 특화 규칙 존재

### 민사서류
- 소장 작성 형식: 당사자/청구취지/청구원인/입증방법
- 등기부등본/가족관계증명서 표 데이터 → AI 줄글 재구성 필요
- 법률 논증(시효중단, 대위청구 등)은 TTS 적합

### 부동산등기서류
- 등기신청서 양식이 핵심 → 표/2단배치/빈칸 처리 난관
- 첨부서면 해설 부분은 서술형으로 TTS 적합
- 현재 보류 상태 (사용자 판단)
