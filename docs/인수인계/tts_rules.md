# TTS 전처리 규칙 (확정판)

사용자(수험생) 피드백 2차례 반영 완료. 이 문서가 TTS 변환의 최종 규칙.

## 기본 전처리 규칙 (10개)

| # | 규칙 | 예시 |
|---|------|------|
| 1 | 한자 인명 → 한글 | 甲→갑, 乙→을, 丙→병 |
| 2 | **판례번호 완전 제거** | ~~(대판 2018.7.19. 2018다22008)~~ → 삭제 |
| 3 | 날짜 정규화 | 2020. 1. 5. → 2020년 1월 5일 |
| 4 | 페이지 헤더/푸터 제거 | "1/3 법무사형법..." 등 삭제 |
| 5 | 꺽쇠 인용부호 제거 | 「부동산등기법」→ 부동산등기법 |
| 6 | 특수 띄어쓰기 정리 | "근 거" → "근거" |
| 7 | 한자 병기 제거 | 구법(舊法) → 구법 (인명 甲乙丙은 한글 변환) |
| 8 | 원문자 번호 통일 | ① → 첫째 또는 1번 |
| 9 | O/x 표기 변환 | 경우-O → 인정됨, 경우-x → 부정됨 |
| 10 | 빈칸(○○○) 처리 | ○○○원 → (금액)원 또는 정답 삽입 |

## 사용자 피드백 반영 추가 규칙 (11개)

### R-01. 판례번호는 변환이 아닌 완전 제거
- **기존**: 판례번호를 자연어로 변환 (대판 2018.7.19. → 대법원 2018년 7월 19일 판결)
- **확정**: 판례번호 자체를 완전히 제거. TTS에서 불필요한 정보.
- 예: "(대판 2018.7.19. 2018다22008)" → 삭제

### R-02. 날짜 → 상대적 시간 표현 활용
- 날짜가 빈번하면 듣는 사람이 날짜 파악이 불가능
- 문맥상 가능하면 상대적 표현으로 변환
- 예: "2000년 1월 1일 대여, 2005년 3월 소장접수" → "2000년 1월경 대여하고 5년이 지난 이후 소장 접수"
- 단, 시효/기간 계산에 필수인 날짜는 유지

### R-03. 괄호(A(B)) → "A 혹은 B"
- TTS가 괄호를 읽지 못함
- 예: "행위시법주의(소급효금지의 원칙)" → "행위시법주의 혹은 소급효금지의 원칙"
- 보충 설명 괄호도 동일: "채무자(피고)" → "채무자 혹은 피고"

### R-04. 호흡 조절 쉼표 삽입
- 긴 문장은 문맥에 맞게 쉼표(,)를 삽입하여 TTS가 끊어 읽도록 유도
- TTS 엔진은 쉼표(,)와 마침표(.)에서 호흡을 쉼
- 너무 긴 한 문장 → 듣는 사람이 암기 불가
- 반대로 불필요한 위치에 쉼표 남발 → 끊겨 들림
- 듣는 사람의 암기와 이해를 기준으로 판단

### R-05. 콜론(:) → 쉼표(,)
- TTS가 콜론을 읽지 못함
- 예: "결론: 타당하지 않다" → "결론, 타당하지 않다"
- 예: "효과: 증명책임이 전환된다" → "효과, 증명책임이 전환된다"

### R-06. 문제 후 목차 먼저 → 답안
- 문제를 읽은 직후 답안의 목차를 먼저 한번 읽어줌
- 그 후 목차 순서에 맞게 답안을 구성하여 읽음
- 이유: 판례를 못 외워도 목차만큼은 꼭 외워야 함

### R-07. 목차는 원문 그대로
- 목차를 요약하거나 재구성하지 않음
- PDF 원본의 목차 표현을 그대로 사용

### R-08. 특수기호 제거
- TTS가 읽지 못하는 기호들을 제거 또는 변환
- 대상: `[]` `—` `:` `()` `/`
- 대괄호 안 내용은 문맥에 맞게 자연어로 변환
- 예: "[문제]" → "문제" 또는 구분자로 활용

### R-09. AI 자의적 해석 금지 — 원본 정보만 사용
- AI가 법률 지식으로 내용을 보충/해석/변경하면 안 됨
- PDF에 있는 정보만으로 TTS 텍스트 구성
- "TTS 맥락 자연스러움"과 "AI 자의적 해석"의 경계를 명확히
- 의심스러운 경우 원본 표현 유지가 원칙

### R-10. diff 필수 제공
- 매 변환 결과에 원본 vs TTS 비교(diff)를 함께 제공
- 사용자가 최종 검수할 수 있도록

### R-11. 듣는 사람 입장 QA 필수
- 변환 완료 후 아래 관점에서 자체 QA 수행:
  - 이 부분이 구분되어 들리는가?
  - 문장이 너무 길어서 맥락 이해나 암기가 힘들지 않은가?
  - 날짜/인명이 혼동되지 않는가?
  - 법률 용어가 정확히 전달되는가?

## 과목별 특화 전처리

| 과목 | 특화 규칙 |
|------|----------|
| 민소/민법 | 빈칸 박스 처리 전략 (빈칸 유지 vs 정답 삽입 — 선택 필요) |
| 형법/형소 | 별표(★★) → "중요" 변환, 중첩 괄호 단순화 |
| 부등법 | "명문x" → "명문 규정 없음" 변환 |
| 민사서류 | 등기부/가족관계 표 → AI 줄글 재구성 |
| 부등서류 | 보류 (등기신청서 양식 처리 미정) |

## 규칙 적용 우선순위

1. R-09 (자의적 해석 금지) — 최상위. 다른 규칙과 충돌 시 이 규칙이 우선
2. R-06, R-07 (목차 구조) — 답안 구성의 뼈대
3. R-01~R-05, R-08 (텍스트 정제) — 기계적 적용
4. R-04 (호흡 조절) — 최종 단계에서 듣기 관점 조정
5. R-10, R-11 (검증) — 출력 후 필수
