# 무선 디버깅 가이드 (갤럭시 + 맥북)

## 언제 사용하나
- 앱이 번개처럼 꺼지는 크래시 → JS 로그로 안 잡힘 → adb logcat 필요
- USB 케이블 없을 때

## 사전 준비 (1회)
1. 갤럭시 **설정 → 휴대전화 정보 → 빌드번호 7번 탭** → 개발자 옵션 활성화
2. **설정 → 개발자 옵션 → 무선 디버깅 ON**

## 연결 순서

### 1. 페어링 (최초 1회 또는 만료 시)
갤럭시: 무선 디버깅 → "페어링 코드로 기기 페어링"
```bash
adb pair <IP>:<페어링포트> <페어링코드>
# 예: adb pair 10.77.76.239:42167 452143
```

### 2. 연결
무선 디버깅 화면 하단의 "IP 주소 및 포트" 확인 (페어링 포트와 다름!)
```bash
adb connect <IP>:<연결포트>
# 예: adb connect 10.77.76.239:45147
```

### 3. 크래시 로그 확인
```bash
# 기존 로그 클리어
adb -s <IP>:<포트> logcat -c

# 크래시 재현 후 로그 확인
adb -s <IP>:<포트> logcat -d | grep -A 20 "FATAL EXCEPTION"
```

## 주의
- PC와 폰이 **같은 Wi-Fi 네트워크**에 있어야 함
- 페어링 포트 ≠ 연결 포트 (헷갈리지 말 것!)
- 무선 디버깅은 폰 재시작하면 꺼짐 → 다시 켜야 함
